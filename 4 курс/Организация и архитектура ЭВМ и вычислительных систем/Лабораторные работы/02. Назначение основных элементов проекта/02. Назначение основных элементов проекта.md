**Задание: «Создание модуля учета задач»**  
**Цель задания:** Закрепить на практике навыки создания основных элементов проекта и их взаимосвязи путем разработки простого модуля учета задач в рамках приложения.  
**Исходные условия:** В вашем проекте уже создана подсистема `Основное`. Вам необходимо доработать проект, руководствуясь следующим сценарием.  
**Часть 1: Создание элементов проекта**  
1. Пакет `CRM`
	- в подсистеме Основное создайте пакет с именем `CRM`
2. Справочник `Исполнители`
	- Создайте в пакете `CRM` справочник с именем `Исполнители`
	- Добавьте к нему реквизит `Телефон` (тип: `Строка`) и `ЭлектроннаяПочта` (тип: `Строка`)
3. Перечисление `СтатусыЗадач`
	- Создайте перечисление с именем `СтатусыЗадач`
	- Добавите в него предопределенные значения: `Новая`, `ВРаботе`, `Выполнена`
	- Установите значение `Новая` значением по умолчанию.
4. Документ `Задача`
	- Создайте документ с именем `Задача`
	- Добавьте к документу следующие реквизиты:
		- `Тема` (тип: `Строка`) - краткое описание задачи
		- `Исполнитель` (тип: `Исполнители.Ссылка`) - ссылка на справочник `Исполнители`. Не забудьте о возможности значения `Неопределено`
		- `Статус` (тип: `СтатусыЗадач`) - статус задачи из перечисления
  
**Часть 2: Организация интерфейса**
1. Настройка интерфейса
	- Убедитесь, что для всех созданных объектов в свойствах установлена `ОбластьВидимости` -> `ВПроекте`
	- В подсистеме `СтандартноеПриложение` откройте настройки `ОсновноеПриложение`
	- Добавьте в панель навигации новую группу `Задачи`
	- Внутри группы создайте навигационные команды для справочника `Исполнители` и документа `Задача`
  
**Часть 3: Проверка работы (Тестирование)**
1. Заполнение данными и проверка
	- Опубликуйте проект и запустите его в пользовательском режиме
	- Перейдите в раздел `Задачи` -> `Исполнители` и создайте 2-3 новых исполнителя, заполнив их наименование и контактную информацию
	- Перейдите в раздел `Задачи` -> `Задача` и создайте 3-4 новых документа `Задача`. Для каждой задачи укажите тему, выберите созданного исполнителя и посмотрите, как проставляется статус по умолчанию
  
**Что должно получиться:** в итоге у вас будет простое, но рабочее приложение, где можно заводить исполнителей и ставить им задачи, отслеживая их статус. Это основа для будущей более сложной системы управления задачами.  
**Результат для сдачи:** представьте преподавателю гиперссылку на ваше работающее приложение и логин/пароль тестового пользователя (созданного в разделе `Пользователи` карточки приложения), чтобы он мог проверить работу механизма.
